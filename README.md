# Визуализатор алгоритмов на графах (вариант А-03)

Учебная ознакомительная практика, ФГБОУ ВО «ВВГУ», Институт информационных
технологий и анализа данных, кафедра информационных технологий и систем,
группа БИС-24-3.

Интерактивный веб-инструмент для построения графов и пошагового
выполнения на них алгоритмов обхода (BFS, DFS), поиска кратчайших путей
(алгоритм Дейкстры) и построения минимального остовного дерева
(алгоритмы Прима и Краскала).

## Возможности

- Интерактивный редактор графа: добавление вершин кликом, добавление
  рёбер с указанием веса, удаление вершин и рёбер через контекстное меню.
- Пошаговое выполнение выбранного алгоритма с подсветкой текущей
  рассматриваемой вершины, текущего ребра и рёбер, вошедших в результат.
- Управление воспроизведением: шаг вперёд / назад, автоматическое
  проигрывание с регулируемой скоростью.
- Вывод результата: порядок обхода (BFS/DFS), таблица расстояний
  (Дейкстра), список рёбер и суммарный вес остова (Прим/Краскал).
- Импорт и экспорт графа в формате JSON, готовые тестовые наборы данных.

## Архитектура

Проект разделён на слои (см. раздел 3 отчёта):

| Путь | Назначение |
|---|---|
| `src/core/graph.py` | модель графа: список смежности, сериализация/десериализация JSON, генерация случайных графов |
| `src/core/steps.py` | единый формат шагов выполнения: классы `Step` и `StepRecorder` |
| `src/core/algorithms/traversal.py` | BFS, DFS |
| `src/core/algorithms/dijkstra.py` | алгоритм Дейкстры (на основе `heapq`) |
| `src/core/algorithms/mst.py` | алгоритмы Прима и Краскала (с DSU) |
| `src/server/app.py` | серверная часть на Flask, REST-подобный API |
| `src/web/` | пользовательский интерфейс: HTML, CSS, JavaScript, отрисовка на Canvas |
| `tests/` | автоматизированные модульные тесты на pytest + тестовые данные |
| `Dockerfile`, `requirements.txt` | файлы контейнеризации и список зависимостей |

Алгоритмическое ядро полностью независимо от Flask и веб-интерфейса:
модули `traversal.py`, `dijkstra.py`, `mst.py` можно подключать и
тестировать без поднятия сервера.

## Установка и запуск локально (без Docker)

Требуется Python 3.10+.

```bash
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

# запуск тестов
python3 -m pytest tests/ -v

# запуск сервера
python3 -m src.server.app
```

Открыть в браузере: **http://localhost:5000**

## Запуск в Docker

Требуется установленный Docker (Desktop для Windows/macOS или Docker
Engine для Linux).

### 1. Сборка образа

В корневой директории проекта (там, где лежит `Dockerfile`) выполнить:

```bash
docker build -t graph-viz .
```

### 2. Запуск контейнера

```bash
docker run -p 5000:5000 graph-viz
```

Флаг `-p 5000:5000` пробрасывает порт контейнера на хост-машину.

### 3. Открыть интерфейс

Перейти в браузере на **http://localhost:5000**.

### Полезные команды

```bash
# запуск в фоновом режиме (detached)
docker run -d -p 5000:5000 --name graph-viz graph-viz

# просмотр логов фонового контейнера
docker logs -f graph-viz

# остановка и удаление контейнера
docker stop graph-viz && docker rm graph-viz

# если порт 5000 уже занят — пробросить на другой порт хоста
docker run -p 5050:5000 graph-viz   # тогда открывать http://localhost:5050

# запуск тестов внутри собранного образа
docker run --rm graph-viz python3 -m pytest tests/ -v
```

### Типичные проблемы при запуске (см. раздел 6 отчёта)

| Проблема | Решение |
|---|---|
| Порт 5000 уже занят другим процессом | пробросить контейнер на другой свободный порт хоста, например `-p 5050:5000` |
| Браузер показывает пустой холст | убедиться, что контейнер полностью запущен (`docker logs graph-viz`), обновить страницу через несколько секунд |
| Ошибка при импорте JSON-файла графа | проверить соответствие файла формату экспорта продукта |
| Алгоритм Дейкстры не запускается | убедиться, что все веса рёбер графа неотрицательны |
| Контейнер не собирается | проверить версию установленного Docker (`docker --version`) и наличие свободного места на диске |

## Использование интерфейса

1. **Редактирование графа.** Клик левой кнопкой мыши по пустой области
   холста создаёт новую вершину. Инструмент «Добавить ребро» + клик по
   двум вершинам создаёт связь между ними (будет предложено ввести вес,
   по умолчанию `1.0`). Удаление вершины или ребра — через контекстное
   меню (клик правой кнопкой мыши).
2. **Запуск алгоритма.** В боковой панели выбрать алгоритм и стартовую
   вершину (не требуется для Краскала), нажать «Старт».
3. **Воспроизведение.** Кнопки «шаг вперёд / назад» или автоматическое
   проигрывание с регулировкой скорости ползунком.
4. **Результат.** В нижней части боковой панели отображается итог:
   порядок обхода, таблица расстояний или список рёбер остова с
   суммарным весом.
5. **Импорт/экспорт.** Кнопка «Экспорт в JSON» сохраняет текущий граф;
   «Импорт» загружает ранее сохранённый файл. Также доступны три готовых
   примера графов (малый, несвязный, случайный на 20 вершин).

## Пример использования ядра напрямую (без веб-интерфейса)

```python
from src.core.graph import Graph
from src.core.algorithms.dijkstra import dijkstra

g = Graph(directed=False)
g.add_edge("A", "B", 1)
g.add_edge("B", "C", 2)

result = dijkstra(g, "A")
print(result["distances"])  # {'A': 0, 'B': 1, 'C': 3}
print(result["steps"])      # список шагов для визуализации
```

## Тестирование

Покрытие тестами по модулям ядра и серверу (см. раздел 5 отчёта):

| Файл | Что проверяется |
|---|---|
| `tests/test_graph.py` | построение графа, сериализация/десериализация JSON, генерация `random_graph` |
| `tests/test_traversal.py` | корректность порядка обхода BFS и DFS, граничные случаи |
| `tests/test_dijkstra.py` | корректность расстояний, обработка несвязных вершин, отрицательные веса отклоняются |
| `tests/test_mst.py` | корректность Прима и Краскала, совпадение суммарного веса, отсутствие циклов |
| `tests/test_steps.py` | согласованность журнала шагов с итоговым результатом алгоритма |
| `tests/test_server.py` | REST API: `/api/run`, `/api/sample/<name>`, валидация входных данных |

```bash
python3 -m pytest tests/ -v
```
